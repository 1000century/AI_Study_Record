🔄 1. Reranking이란?

Reranking은 초기 검색 결과를 더 정확한 관련성 점수로 재정렬하는 과정입니다.

🔧 2. Reranking 코드 구조

1 CrossEncoder 모델 초기화
from sentence_transformers import CrossEncoder # BGE Reranker 모델 로드 reranker = CrossEncoder("BAAI/bge-reranker-v2-m3")
2 Reranking 함수 정의
def rerank_documents(query: str, documents: List[str], top_k: int = 5): # 1. 쿼리-문서 쌍 생성 pairs = [(query, doc) for doc in documents] # 2. 관련성 점수 계산 scores = reranker.predict(pairs) # 3. 점수 기준으로 정렬 doc_scores = list(zip(documents, map(float, scores))) doc_scores.sort(key=lambda x: x[1], reverse=True) return doc_scores[:top_k] # 상위 k개 반환
✅ 핵심 포인트: CrossEncoder는 쿼리와 문서를 함께 입력받아 직접적인 관련성을 계산합니다.

📊 3. 평가 메트릭 설명

🎯 Hit Rate@k
상위 k개 결과 중에 정답이 포함된 비율
공식: (정답 포함 쿼리 수) / (전체 쿼리 수)
🏆 MRR (Mean Reciprocal Rank)
정답이 나타나는 평균적인 역순위
공식: 1/순위의 평균 (높을수록 좋음)
💡 예시: 정답이 3번째에 있다면 MRR = 1/3 = 0.333

🔍 4. 평가 과정 단계별 분석

1 데이터 준비
# 평가용 샘플 추출 eval_samples = random.sample(list(evaluation_data), 20) # 각 샘플 구조: # { # 'query': '질문', # 'answer': '정답', # 'relevant_docs_metadata': [관련문서정보] # }
2 원본 검색 수행
# 벡터 유사도 기반 초기 검색 retrieved_docs = retriever.get_relevant_documents(query) original_docs = [doc.page_content for doc in retrieved_docs]
3 Reranking 적용
# CrossEncoder로 재순위화 reranked_results = reranker_func(query, original_docs, top_k=10) reranked_docs = [doc for doc, _ in reranked_results]
4 성능 비교 평가
# Hit Rate@k 계산 for k in [1, 3, 5, 10]: hit_rate_original = calculate_hit_rate(ground_truth, original_docs, k) hit_rate_reranked = calculate_hit_rate(ground_truth, reranked_docs, k) improvement = hit_rate_reranked - hit_rate_original

🧠 5. Semantic Similarity 기반 평가

⚠️ 기존 문제점: 단순 문자열 매칭으로는 의미적으로 같지만 표현이 다른 답변을 놓칠 수 있습니다.
🔬 개선된 평가 방법
def calculate_semantic_hit_rate(ground_truth, retrieved_docs, embeddings_model, threshold=0.7): # 1. 임베딩 생성 gt_embedding = embeddings_model.embed_query(ground_truth) doc_embeddings = embeddings_model.embed_documents(retrieved_docs[:k]) # 2. 코사인 유사도 계산 similarities = cosine_similarity([gt_embedding], doc_embeddings)[0] # 3. 임계값 이상이면 Hit return 1.0 if max(similarities) >= threshold else 0.0
✅ 장점: 의미적으로 유사한 답변도 올바르게 평가할 수 있습니다.

💡 8. 핵심 인사이트

🚀 Reranking의 효과
  • 빠른 벡터 검색으로 후보 문서 확보
  • 정밀한 CrossEncoder로 관련성 재평가
  • 특히 상위 랭킹에서 큰 개선 효과
📊 평가 메트릭의 의미
  • Hit Rate@1: 첫 번째 결과의 정확도
  • Hit Rate@5: 상위 5개 중 정답 포함률
  • MRR: 평균적인 정답 발견 속도
🎯 실무 적용 팁
  • 의료/법률 등 정확성이 중요한 도메인에 효과적
  • 초기 검색 결과가 많을 때 더 큰 효과
  • 계산 비용과 성능 향상의 트레이드오프 고려
